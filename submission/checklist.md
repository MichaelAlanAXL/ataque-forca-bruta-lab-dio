# Checklist de Submissão - Bootcamp

Use este checklist antes de enviar o link do repositório para avaliação.

- [ ] Link do repositório (GitHub): https://github.com/MichaelAlanAXL/ataque-forca-bruta-lab-dio
- [ ] `README.md` com resumo, objetivos e aviso legal
- [ ] `docs/relatorio_final.md` ou PDF com o relatório completo
- [ ] `docs/evidencias.md` (versão texto ou imagens)
- [ ] Evidências em `evidences/logs/` (logs sanitizados)
- [ ] Evidências em `evidences/screenshots/` (opcional)
- [ ] `vm-setup/README.md` descrevendo configurações das VMs
- [ ] `docs/guia_medusa.md` com instruções seguras de execução
- [ ] `docs/mitigacoes.md` com recomendações práticas (inclui trechos de código)
- [ ] `.gitignore` configurado para não subir arquivos sensíveis
- [ ] Licença (ex.: LICENSE)
- [ ] Checklist preenchido e salvo em `submission/checklist.md`

Comentários:
- Se optar por repo público, confirme que não há dados sensíveis.
- Se usar repo privado, verifique se o avaliador tem acesso.
